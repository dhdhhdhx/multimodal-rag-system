# 操作系统 - 软件设计师考试复习指南

## 一、进程管理

### 1.1 进程的状态与转换

进程的五种基本状态：

| 状态 | 描述 |
|------|------|
| 就绪态 | 已获得除CPU外的所有资源 |
| 运行态 | 正在CPU上执行 |
| 阻塞态 | 等待某个事件完成 |
| 创建态 | 正在创建过程中 |
| 终止态 | 进程结束，回收资源 |

状态转换规则：
- 就绪 → 运行：调度程序选中（获得CPU）
- 运行 → 就绪：时间片用完 / 被高优先级进程抢占
- 运行 → 阻塞：等待I/O操作 / 请求资源
- 阻塞 → 就绪：等待的事件发生

### 1.2 进程同步与互斥

**PV操作（信号量机制）**：

```
P(S)：  S = S - 1
        若 S < 0，进程阻塞，进入等待队列
        若 S ≥ 0，进程继续执行

V(S)：  S = S + 1
        若 S ≤ 0，从等待队列唤醒一个进程
        若 S > 0，进程继续执行
```

**经典同步问题**：

1. 生产者-消费者问题
2. 读者-写者问题
3. 哲学家就餐问题

### 1.3 死锁

死锁的四个必要条件：
1. **互斥条件**：资源同时只能被一个进程使用
2. **保持和等待**：进程持有资源同时等待其他资源
3. **不剥夺条件**：已分配资源不能被强行收回
4. **循环等待**：存在进程的循环等待链

死锁处理策略：
- 预防：破坏四个必要条件之一
- 避免：银行家算法
- 检测与恢复：资源分配图化简
- 鸵鸟策略：忽略死锁（Unix常用）

> **考试高频考点**：银行家算法的安全性序列判断、死锁条件分析

## 二、存储管理

### 2.1 分页存储

- 将逻辑地址分为：页号 + 页内偏移
- 页表：页号 → 物理块号的映射
- 地址转换过程：逻辑地址 → 页号查页表 → 物理块号 + 偏移 = 物理地址

快表（TLB）：
- 加速地址转换，命中率越高，平均访问时间越短
- 平均访问时间 = h × (访问TLB + 访问内存) + (1-h) × (访问页表 + 访问内存)

### 2.2 分段存储

- 按逻辑分段，每段有段号和段内偏移
- 便于共享和保护
- 段表：段号 → 段基址 + 段长

### 2.3 段页式存储

- 先分段再分页
- 逻辑地址：段号 + 段内页号 + 页内偏移
- 需要两次查表

### 2.4 页面置换算法

| 算法 | 描述 | 备注 |
|------|------|------|
| FIFO | 先进先出 | 可能产生Belady异常 |
| LRU | 最近最少使用 | 性能好，开销大 |
| OPT | 最佳置换 | 理论最优，无法实现 |
| LFU | 最不经常使用 | 基于访问频率 |

## 三、磁盘管理

### 3.1 磁盘调度算法

常见调度算法比较：

- **FCFS（先来先服务）**：按请求顺序访问，公平但效率低
- **SSTF（最短寻道时间优先）**：选择离当前磁头最近的请求，可能产生饥饿
- **SCAN（电梯算法）**：磁头沿一个方向移动，到达端点后反向，不会产生饥饿
- **C-SCAN（循环扫描）**：单向扫描，到达端点后快速回到起点，各位置等待时间更均匀

### 3.2 磁盘结构

- 寻道时间：磁头移动到目标磁道的时间
- 旋转延迟：磁盘旋转到目标扇区的时间
- 数据传输时间：读写数据的时间

磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间

## 四、文件管理

### 4.1 文件目录结构

- 单级目录：所有文件在同一目录
- 两级目录：按用户分组
- 树形目录：多级层次结构
- 无环图目录：支持文件共享

### 4.2 文件存储

连续分配 → 链接分配 → 索引分配

索引节点（inode）：
- 存放文件的属性信息和数据块指针
- 包含直接块指针和间接块指针
